# SMS Environment Variable Manager

WPF-приложение на платформе **.NET 8**, разработанное в рамках тестового задания SmartMealService.

Приложение предоставляет графический интерфейс для просмотра и изменения набора переменных среды Windows, список которых задается в конфигурационном файле `appsettings.json`.

---

## Возможности

- Просмотр списка переменных среды
- Редактирование значений переменных
- Создание отсутствующих переменных автоматически при запуске
- Сохранение изменений в пользовательские переменные среды Windows (`EnvironmentVariableTarget.User`)
- Логирование всех изменений в файл
- Настраиваемый список переменных через `appsettings.json`

## Конфигурация

Список переменных среды задается в файле `appsettings.json`.

Пример:

```json
{
  "EnvironmentVariables": [
    "SOME_TEST_VAR_1",
    "SOME_TEST_VAR_2",
    "SOME_API_KEY",
    "SOME_CONNECTION_STRING"
  ]
}
```

При запуске приложение:

1. считывает список имен переменных;
2. проверяет их наличие;
3. при отсутствии автоматически создает переменную со значением по умолчанию (`String.Empty`);
4. отображает значения в таблице.

---

## Логирование

Все изменения переменных среды записываются в каталог:

```
Logs/
```

Имя файла соответствует требованиям задания:

```
test-sms-wpf-app-yyyyMMdd.log
```

Пример:

```
Logs/
└── test-sms-wpf-app-20260721.log
```

Пример записи:

```
[INF] Variable 'SMS_API_URL' changed.
Old value: http://localhost
New value: http://192.168.0.5
```

---

## Хранение переменных

Изменения сохраняются в пользовательские переменные среды Windows:

```csharp
EnvironmentVariableTarget.User
```

Благодаря этому:

- изменения доступны другим приложениям;
- сохраняются после перезагрузки Windows;
- не требуют запуска приложения с правами администратора.

## Архитектура

Проект построен с разделением ответственности.

- **MainWindow** — отвечает только за пользовательский интерфейс.
- **EnvironmentVariableService** — загрузка и сохранение переменных среды.
- **EnvironmentConfiguration** — чтение конфигурации приложения.
- **Serilog** — логирование изменений.

Такое разделение упрощает сопровождение и дальнейшее расширение приложения.
