# SmartMealService Test Task (.NET 8)

Репозиторий содержит решение тестового задания для **SmartMealService**, выполненное на платформе **.NET 8**.

В рамках задания реализованы:

- HTTP-клиент для взаимодействия с API;
- gRPC-клиент, реализующий аналогичный функционал;
- консольное приложение для демонстрации работы клиентов;
- WPF-приложение для управления переменными среды Windows.

---

# Структура решения

```
src
│
├── Sms.ConsoleApp
├── Sms.GrpcClientLibrary
├── Sms.HttpClientLibrary
├── Sms.Models
└── Sms.WpfEnvManager
```

## Sms.ConsoleApp

Консольное приложение для демонстрации работы библиотек.

Функции:

- получение меню;
- отправка заказа;
- выбор используемого транспорта (HTTP / gRPC);
- использование параметров из `appsettings.json`.

---

## Sms.HttpClientLibrary

Реализация **Части 1** тестового задания.

Используется:

- HttpClient;
- Basic Authentication;
- JSON Serialization;
- асинхронные запросы (`async/await`).

Основные возможности:

- получение меню;
- отправка заказа;
- обработка ошибок API;
- поддержка `CancellationToken`.

---

## Sms.GrpcClientLibrary

Реализация **Части 2** тестового задания.

Используется:

- gRPC;
- Protocol Buffers v3;
- Grpc.Net.Client;
- Google.Protobuf.

Основные возможности:

- получение меню;
- отправка заказа;
- преобразование protobuf-моделей в доменные модели;
- обработка ошибок gRPC.

---

## Sms.Models

Общее библиотека моделей, используемая HTTP- и gRPC-клиентами.

Содержит:

- Dish
- Order
- OrderItem
- модели ответов API

Использование общей библиотеки позволяет избежать дублирования кода.

---

## Sms.WpfEnvManager

Реализация **Задания №2**.

Возможности приложения:

- просмотр переменных среды;
- изменение значений;
- автоматическое создание отсутствующих переменных;
- сохранение в пользовательские переменные среды Windows (`EnvironmentVariableTarget.User`);
- логирование изменений;
- загрузка списка переменных из `appsettings.json`.

Используемые библиотеки:

- WPF
- Serilog
- Microsoft.Extensions.Configuration

---

# Конфигурация

Консольное приложение использует настройки из файла

```
Sms.ConsoleApp/appsettings.json
```

Пример:

```json
{
  "ApiSettings": {
    "BaseUrl": "https://localhost:5001/",
    "Username": "user",
    "Password": "password"
  }
}
```

Настройки содержат:

- адрес API;
- учетные данные для HTTP Basic Authentication;
- параметры логирования.

# Архитектура

Решение разделено на независимые проекты.

```
                 +----------------+
                 | Sms.ConsoleApp |
                 +-------+--------+
                         |
          +--------------+--------------+
          |                             |
+---------v---------+         +---------v---------+
| HttpClientLibrary |         | GrpcClientLibrary |
+---------+---------+         +---------+---------+
          |                             |
          +-------------+---------------+
                        |
                 +------v------+
                 | Sms.Models  |
                 +-------------+

+----------------------+
| Sms.WpfEnvManager    |
+----------------------+
```

Такое разделение позволяет повторно использовать модели и независимо развивать различные способы взаимодействия с сервером.
